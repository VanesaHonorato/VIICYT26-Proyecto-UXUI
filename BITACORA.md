# Bitácora del Proyecto

## Semana 1
**Periodo:** 15 - 19 de junio

### Objetivo
Configurar el entorno de trabajo y preparar las herramientas necesarias para el desarrollo del proyecto.

---

## Actividades realizadas

### Configuración del repositorio
- Se creó el repositorio del proyecto en GitHub.
- Se agregaron los integrantes del equipo como colaboradores.

### Configuración del entorno
- Instalación y configuración de Git.
- Instalación de Node.js.
- Configuración de Claude Desktop.

### Configuración de MCP
- Se configuró el servidor MCP.
- Se realizaron pruebas de comunicación entre Claude y Figma.
- Se verificó que Claude pudiera interactuar con archivos de Figma mediante MCP.

### Avances
- Se revisó la propuesta del proyecto.
- Se discutieron las ideas iniciales para el diseño de la plataforma.

---

## Observaciones

Durante esta semana se dejó preparado el entorno de desarrollo para comenzar con el diseño de la interfaz en las siguientes etapas del proyecto.


---


## Semana 2

**Periodo:** 22 - 26 de junio

## Objetivo

Investigar el proceso de diseño UX para definir la estructura de navegación de la plataforma y establecer las bases para el desarrollo de la interfaz de usuario.

## Actividades realizadas

- Se investigaron los conceptos básicos de User Experience (UX).
- Se utilizó Claude como herramienta de apoyo para la generación de propuestas de diseño.
- Se redactó un prompt para generar la estructura del Sitemap considerando los diferentes tipos de usuarios y las funcionalidades del sistema.
- Se obtuvo un Sitemap inicial, el cual fue revisado y adaptado por el equipo.
- A partir del Sitemap se desarrolló el User Flow de la plataforma.
- Se realizaron wireframes de baja fidelidad en el pizarrón para visualizar la distribución de las principales pantallas y módulos del sistema.
- Se discutió en equipo la arquitectura de navegación y la organización de las pantallas.

## Evidencias

- Sitemap.
- User Flow.
- Wireframes de baja fidelidad.
- Prompt utilizado en Claude.

## Observaciones

Durante esta semana se definió la estructura de navegación de la plataforma y se establecieron las bases para el diseño de la interfaz. El uso de Claude permitió agilizar la generación de propuestas que posteriormente fueron revisadas y ajustadas por el equipo.

---

## Semana 3
**Periodo:** 29 de junio - 3 de julio

### Objetivo
Diseñar el Sistema de Diseño de la plataforma, definir la estructura de navegación y preparar las herramientas de inteligencia artificial para la generación de wireframes mediante Figma, Stitch y Antigravity.

---

## Actividades realizadas

### Reunión de asesoría
- Se aclararon dudas sobre el proyecto.
- Se revisó el objetivo general de la plataforma.
- Se acordó diseñar una plataforma con funcionalidades generales para facilitar su escalabilidad.

### Diseño del Sitemap
- Se comenzó la elaboración del mapa del sitio de la plataforma.
- Se identificaron las pantallas correspondientes a los roles de Alumno y Coordinador.
- Se organizó la navegación entre las diferentes secciones del sistema.

### Planeación del Sistema de Diseño
- Se inició la definición de la identidad visual de la plataforma.
- Se investigó la psicología del color para seleccionar una paleta adecuada para una plataforma educativa.
- Se definió la tipografía principal del proyecto.
- Se establecieron los componentes base que formarán parte del Sistema de Diseño:
  - Botones
  - Inputs
  - Cards
  - Header
  - Sidebar
  - Iconografía

### Investigación de herramientas
- Se investigó el funcionamiento de Google Stitch para la generación de interfaces mediante IA.
- Se revisó el uso de Antigravity como apoyo para el diseño de interfaces.
- Se analizó el funcionamiento del protocolo MCP para conectar las herramientas con Figma.

### Configuración del entorno
- Se generó la API Key de Stitch.
- Se inició la configuración del servidor MCP.
- Se realizaron pruebas para integrar Stitch, Antigravity y Figma.
- Se preparó el entorno para comenzar la generación automática de wireframes.

### Avances
- Se obtuvo una estructura inicial del Sitemap.
- Se definió la base del Sistema de Diseño.
- Se preparó el entorno para utilizar herramientas de inteligencia artificial en el proceso de diseño.

---

## Observaciones

Durante esta semana se definió la dirección visual del proyecto y se resolvieron las principales dudas relacionadas con el alcance de la plataforma. Además, se inició la configuración de las herramientas que permitirán automatizar la creación de wireframes y componentes durante las siguientes etapas del proyecto.
